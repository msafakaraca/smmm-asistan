- Hizmetler = mükelleflere ek hizmet faturalandırma

- Mevcut `/api/finance/transactions` POST kullanılabilir
- SMM hesaplaması gerekli (SMMCalculator reuse)
- Müşteri seçimi zorunlu
- Kategori: INCOME tipinde

### Giderler Mantığı (E3-S2)
- Giderler = ofis giderleri (mükelefe BAĞLI DEĞİL)
- Data: `expenses` tablosu (ayrı model)
- Mevcut `/api/finance/expenses` CRUD hazır
- SMM hesaplaması YOK
- Müşteri seçimi YOK
- Kategori: EXPENSE tipinde
- Tekrarlayan gider seçeneği (isRecurring + recurringFrequency)

### Tekrarlayan Gider (E3-S3)
- Mevcut expenses modelinde isRecurring ve recurringFrequency alanları VAR
- RecurringFrequency: REC_MONTHLY, REC_QUARTERLY, REC_ANNUAL
- E3-S2 içinde entegre edilecek (ayrı component olarak)
- pg_cron/API-level cron sonraya bırakılabilir (backend otomasyon)

## Teknik Notlar

### Hizmetler → Transactions API Kullanımı
```typescript
// Hizmet oluşturma (POST /api/finance/transactions)
{
  customerId: "...",
  categoryId: "...",
  type: "DEBIT",
  amount: brütTutar,
  grossAmount: brütTutar,
  kdvAmount: kdvTutarı,
  stopajAmount: stopajTutarı,
  netAmount: netTutar, // Brüt + KDV - Stopaj
  date: "2026-02-12",
  dueDate: "2026-03-12", // Opsiyonel
  status: "PENDING",
  description: "Ek hizmet açıklaması",
  autoGenerated: false, // Manuel hizmet
  costDefinitionId: null // Maliyet kalemiyle ilişkili değil
}
```

### Hizmetleri Transactions'dan Ayırt Etme
- Hizmetler: costDefinitionId=null AND autoGenerated=false AND type=DEBIT
- Otomatik borçlar: autoGenerated=true AND costDefinitionId!=null
- Tahsilatlar: type=CREDIT
- Filtre: useServices hook'unda API'ye costDefinitionId=null&autoGenerated=false&type=DEBIT gönderilecek
- **NOT:** Mevcut transactions API'sinde autoGenerated ve costDefinitionId filtresi YOK
  - Çözüm: Ya API'ye filtre ekle, ya da client-side filtrele
  - Tercih: API'ye filtre ekle (performans için)

### Transactions API Filtre Eklenmeli
`src/app/api/finance/transactions/route.ts` GET handler'ına ekle:
```typescript
const autoGenerated = searchParams.get("autoGenerated");
const hasCostDefinition = searchParams.get("hasCostDefinition");

if (autoGenerated !== null) where.autoGenerated = autoGenerated === "true";
if (hasCostDefinition === "false") where.costDefinitionId = null;
if (hasCostDefinition === "true") where.costDefinitionId = { not: null };
```

```typescript
export const serviceFormSchema = z.object({
  customerId: z.string().uuid("Geçerli bir müşteri seçiniz"),

  stopajRate: z.number().min(0).max(100).optional().nullable(),

  dueDate: z.string().optional().nullable(),
  description: z.string().optional(),
});
export type ServiceFormValues = z.infer<typeof serviceFormSchema>;
```

### Dikkat Edilecek Edge Case'ler
- Gider silindiğinde confirmation dialog göster
- Tekrarlayan gider silindiğinde "sadece bu kaydı mı, tüm tekrarları mı?" sorusu şimdilik gereksiz (her kayıt bağımsız)
- Hizmet düzenlendiğinde SMM hesaplaması yeniden yapılmalı
- Para birimi TRY dışında seçildiğinde kur bilgisi (şimdilik opsiyonel, E5'te detaylı)
- Gider kategorileri EXPENSE tipinde filtrelenmeli
- Hizmet kategorileri INCOME tipinde filtrelenmeli

## Kararlar ve Gerekçeler

| Karar | Neden | Alternatifler |
|-------|-------|---------------|
| Hizmetler → transactions tablosu | Borç/alacak takibi, tahsilat entegrasyonu | Ayrı tablo (gereksiz duplicasyon) |
| Giderler → expenses tablosu | Mükelef bağımsız, farklı yapı | transactions tablosu (overload) |
| API filtre ekleme | Client-side filtre performans kaybı | Client-side (veri az olsa da) |
| Tekrarlayan cron → sonra | E3-S3 scope'u daralt, önce UI | Hemen implement (karmaşık) |
| SMMCalculator reuse | Zaten var, test edilmiş | Yeni component (gereksiz) |
